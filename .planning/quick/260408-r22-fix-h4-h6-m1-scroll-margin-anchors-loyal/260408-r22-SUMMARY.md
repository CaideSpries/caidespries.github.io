# Phase 260408-r22 Plan 01: Fix H4 H6 M1 — Scroll Margin, Anchors, Loyalty, Dark Mode Summary

Three targeted UX bug fixes: scroll-margin on all homepage section anchors, LoyaltySection year-gate removal, and Tailwind v4 dark-mode variant reconfiguration so the theme toggle icon tracks the `.dark` class.

## Fixes Applied

### H4 — Scroll margin on section anchors (Task 1)

Added `scroll-mt-20` to the root `<section id="...">` element in all seven homepage sections. The fixed header is `h-16` (64px); `scroll-mt-20` (80px) provides 16px of breathing room so headings are never obscured when navigating via hash links.

Files modified: About.astro, Experience.astro, Ventures.astro, Projects.astro, Education.astro, Skills.astro, Listening.astro.

Commit: `b752bcd`

### H6 — LoyaltySection for any wrapped year with loyalty data (Task 2)

Removed `data.year === new Date().getFullYear() &&` from the LoyaltySection render condition in `src/pages/wrapped/[year].astro`. The `data.loyalty &&` guard is retained so the component only renders when the field is present. Historical wrapped years with loyalty data will now display the section correctly.

Commit: `04fdf81`

### M1 — Dark mode toggle icon responding to .dark class (Task 3)

Root cause: Tailwind v4 defaults the `dark:` variant to `prefers-color-scheme` media query. The site applies a `.dark` class to `<html>` via JavaScript (ThemeToggle + ApplyColorMode). The toggle SVGs use `dark:block` / `dark:hidden`, which responded to the OS media query instead of the class, so the icon ignored the JS toggle entirely.

Fix: Added `@variant dark (&:where(.dark, .dark *));` on line 3 of `src/styles/global.css` (immediately after `@import "tailwindcss";`). This overrides the variant so all `dark:` utilities match the `.dark` class on any ancestor element, matching how the site's theme system works.

No changes were needed to ThemeToggle.astro — the SVG assignments (`hidden dark:block` for sun, `block dark:hidden` for moon) were already correct.

Commit: `7306f70`
