<task type="auto">
  <name>Task 3: Fix dark mode variant so toggle icon responds to .dark class (M1)</name>
  <files>src/styles/global.css</files>
  <action>
    Root cause: Tailwind v4 defaults dark mode to `prefers-color-scheme` media query. The site uses a `.dark` class on `<html>` toggled by JavaScript. The ThemeToggle SVGs use `dark:block` / `dark:hidden` which currently respond to the OS media query, not the class, so the icon ignores the toggle.

    Fix: Add a `@variant` override in global.css immediately after `@import "tailwindcss";` to make Tailwind v4's `dark:` variant match the `.dark` class:

    ```css
    @import "tailwindcss";

    @variant dark (&:where(.dark, .dark *));
    ```

    Insert the `@variant dark (&:where(.dark, .dark *));` line as line 3 (after the import, before the @font-face block). Do not remove or reorder any existing rules.

    The ThemeToggle SVG assignments are already correct:
    - First SVG (sun): `hidden dark:block` — hidden in light, visible in dark. Correct.
    - Second SVG (moon): `block dark:hidden` — visible in light, hidden in dark. Correct.
    No changes needed to ThemeToggle.astro itself.

    Commit:
    `fix(theme): configure Tailwind v4 dark variant to use .dark class, fixing toggle icon`
  </action>
  <verify>
    `grep -n "@variant dark" src/styles/global.css`
    Must output: `@variant dark (&:where(.dark, .dark *));`
    Build passes: `npm run build 2>&1 | tail -5`
  </verify>
  <done>global.css contains @variant dark class selector override on line 3. Build passes. Dark mode toggle icon will now correctly show sun when .dark class is present and moon when absent.</done>
