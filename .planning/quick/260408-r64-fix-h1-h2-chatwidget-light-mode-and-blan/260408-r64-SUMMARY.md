# Phase 260408-r64 Plan 01: ChatWidget Light Mode + Mobile Gaps Summary

Theme-aware ChatWidget popup via CSS classes, reduced section padding from py-24 to py-16 md:py-24, and footer-spacer cut from 50vh to 15vh.

## What Was Done

### Task 1: ChatWidget theme-aware styling

The ChatWidget had hardcoded dark hex values as inline `style` props on the panel container, assistant message bubbles, loading dots bubble, input field, and input wrapper border. In light mode this produced a black popup on a cream page.

Added four CSS classes to `src/styles/global.css` inside `@layer components`:

- `.chat-widget-panel` — dark bg in dark mode, cream (`#faf6f1`) in light mode
- `.chat-widget-assistant-bubble` — translucent cream tint with cream text in dark; translucent dark tint with dark text in light
- `.chat-widget-input` — matching background/border/color per mode, with muted placeholder in light mode
- `.chat-widget-divider` — border-top that adapts per mode

All four use `html:not(.dark)` overrides, matching the existing `.retro-card` pattern already in global.css.

Updated `ChatWidget.tsx` to apply these classes. Ember-colored elements (toggle button, header bar, user message bubbles, send button) kept as inline styles — intentional brand color, correct in both modes.

Commit: `b45041b`

### Task 2: Mobile section padding and footer-spacer

All four content sections (About, Experience, Ventures, Projects) used `py-24` on all breakpoints — 96px top + bottom = 192px per section on mobile. Changed to `py-16 md:py-24`, reducing mobile padding to 64px per side (128px per section), saving 256px total across four sections. Desktop unchanged.

The `.footer-spacer` rule in global.css was `height: 50vh`. On a 667px iPhone SE that is 333px of blank cream. Reduced to `height: 15vh` (~100px on mobile), which still provides enough space for the sticky footer reveal effect on desktop.

Commit: `5f64d04`
