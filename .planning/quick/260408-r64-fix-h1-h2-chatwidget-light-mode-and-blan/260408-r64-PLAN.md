**Root cause confirmed from file reads:**
- All four sections use `py-24` (6rem = 96px each side on mobile)
- `footer-spacer` in global.css is `height: 50vh` — on a 667px iPhone that is 333px of blank cream

**Fix 1 — Section padding (four files):**

In each of the four section files, change the opening `<section>` tag padding class:
- About.astro: `class="py-24 scroll-mt-20"` → `class="py-16 md:py-24 scroll-mt-20"`
- Experience.astro: `class="py-24 scroll-mt-20"` → `class="py-16 md:py-24 scroll-mt-20"`
- Ventures.astro: `class="py-24 scroll-mt-20"` → `class="py-16 md:py-24 scroll-mt-20"`
- Projects.astro: `class="py-24 scroll-mt-20"` → `class="py-16 md:py-24 scroll-mt-20"`

This reduces mobile top+bottom padding from 192px to 128px per section — a 64px saving per section,
256px total across four sections. Desktop (md+) is unchanged at py-24.

**Fix 2 — footer-spacer (global.css):**

In `src/styles/global.css` find the `.footer-spacer` rule (currently `height: 50vh`) and change to:

```css
.footer-spacer {
  height: 15vh;
}
```

15vh is enough for the footer reveal effect to work on desktop (the footer is sticky beneath the
main content and reveals on scroll-past). On a 667px phone, 15vh = ~100px instead of 333px.
This also closes issue M12 as a free side-effect.

Do NOT change any other padding, margin, or min-height values. The Hero uses `min-h-screen` but
that is intentional (full-screen hero) and must not be touched.
