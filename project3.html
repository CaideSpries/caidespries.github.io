Compression and Encryption Algorithm for Communications

Project Overview

This project outlines the requirements and design procedures for the compression and encryption of data received by the Inertial Measurement Unit (IMU) onboard the SHARC buoy in Antarctica. The goal is to compress data to reduce transmission costs and encrypt it for secure transmission.

Objectives

  • Analyse the requirements outlined in the project design brief.
  • Compare various compression and encryption algorithms to determine the most suitable one.
  • Design a system that fulfills the project's requirements for low power usage and high efficiency.
  • Implement the system using the STM32F051 Discovery Board.

Technologies Used

  • STM32F051 Discovery Board
  • Sense HAT B for data collection
  • Twofish encryption algorithm
  • LZSS compression algorithm
  • Python for simulation and testing

Project Details

The project encompasses the following tasks:

  1. Interpretation of user requirements and feasibility analysis.
  2. Comparison of encryption algorithms (AES, RSA, Blowfish, Twofish).
  3. Comparison of compression algorithms (LZ77, LZR, LZSS, Deflate, Brotli).
  4. Design and implementation of the system on the STM32F051 Discovery Board.
  5. Validation and testing using simulated and old data.
  6. Analysis of results and performance evaluation.

Results

The system successfully compressed and encrypted data from the IMU, achieving a compression ratio of at least 1.5 and encryption times within the acceptable range. The implementation of the Twofish and LZSS algorithms was efficient, and the data transmission via Iridium was secure and cost-effective.
